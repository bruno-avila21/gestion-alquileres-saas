        /// <summary>Historical index values by type and date range. IDX-06.</summary>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-18 | Spoofing | Unauthenticated /sync request | mitigate | BaseController applies [Authorize] globally. Unauthenticated call returns 401 (T2 tested). |
| T-02-19 | Elevation of Privilege | Cross-tenant data leak | accept | Indices are GLOBAL by design. Any authenticated user can read them. This is correct per requirements (IDX-06). |
| T-02-20 | Denial of Service | Unbounded GET /indexes query | mitigate | Validator enforces From <= To; frontend sends small ranges. Consider adding a server-side max 24-month guard in future phase if needed. |
| T-02-21 | Denial of Service | Sync endpoint flooded | accept | Manual admin trigger in Phase 2; background Hangfire scheduling is Phase 5. Idempotent (ExistsAsync short-circuit) so repeated POSTs are cheap after first call. If needed, ASP.NET Core rate limiting can be added in API layer without changing Controller. |
| T-02-22 | Tampering | Request body injection (unexpected fields) | mitigate | [FromBody] with record type = strict shape binding. Extra fields are silently ignored by System.Text.Json. FluentValidation rejects invalid enum/period. |
| T-02-23 | Information Disclosure | Error body reveals stack traces | mitigate | ExceptionMiddleware catches unknown exceptions and returns `{ error: "Internal server error" }` — no stack trace. BusinessException message is exposed (intended, user-facing). |
</threat_model>
